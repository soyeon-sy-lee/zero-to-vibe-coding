# Zero to Vibe Coding

> 코딩을 한 번도 해보지 않은 사람이 **Python + Codex + ChatGPT**로 첫 프로그램을 만드는 프로젝트

## 왜 이 프로젝트를 만들었나

처음에는 Codex의 안내를 따라 여러 기능을 구현했지만, 늘 기초 단계를 건너뛰고 중간에서부터 시작한 듯한 느낌이 들었습니다. 프로젝트 폴더가 무엇인지, `cd`는 언제 사용하는지, 터미널·Python·VS Code는 각각 어떤 역할을 하는지 충분히 이해하지 못한 채 AI가 제시한 명령을 따라가고 있었기 때문입니다.

결과물을 만드는 것과 그 과정을 이해하는 것은 다르다는 점을 깨닫고, 개발환경의 기초부터 다시 학습했습니다. 그 과정에서 제가 실제로 막혔던 질문들을 한 단계씩 정리한 것이 이 프로젝트입니다. 목표는 코드를 대신 작성해 주는 튜토리얼이 아니라, 비개발자가 AI가 만든 결과를 직접 실행하고 확인하며 수정할 수 있도록 돕는 것입니다.

> 프로젝트를 만들며 발견한 문제와 학습 방식의 변화는 **[Learning Journey](docs/LEARNING_JOURNEY.md)**에 정리했습니다.

# 🚀 시작하기 전에 (가장 중요!)

이 프로젝트는 책처럼 혼자 읽는 튜토리얼이 아닙니다.

**ChatGPT를 개인 튜터처럼 사용하면서 진행하는 프로젝트**입니다.

## 1. ChatGPT 새 대화를 시작하세요.

## 2. 아래 프롬프트를 그대로 복사해서 붙여넣으세요.

```text
나는 코딩을 한 번도 해본 적 없는 완전 초보자야.

앞으로 너는 내 Python & Codex 튜터 역할을 해줘.

규칙은 다음과 같아.

1. 절대로 한 번에 많은 내용을 설명하지 마.
2. 항상 한 단계씩만 진행해.
3. 내가 실제로 해보고 결과를 보내면 그다음 단계로 넘어가.
4. 내가 이해했는지 계속 확인해.
5. 코드보다 "왜 하는지"를 먼저 설명해.
6. 내가 모른다고 가정하고 설명해.
7. 어려운 용어를 쓰면 반드시 쉽게 비유해서 설명해.
8. 내가 막히면 다른 방법도 제안해.
9. 내가 만든 코드나 Codex 결과는 리뷰해줘.
10. 프롬프트는 처음에는 네가 작성해주고 점점 내가 직접 쓰도록 도와줘.

내 목표는 Python을 배우는 것이 아니라
Codex를 활용해 바이브코딩을 할 수 있는 사람이 되는 것이다.

항상 아래 형식으로 진행해줘.

- 오늘 목표
- 왜 하는지
- 직접 해보기
- 예상 결과
- 자주 하는 실수
- 다음 단계

절대로 여러 단계를 한꺼번에 진행하지 말고,
내가 완료했다고 말하면 다음 단계로 넘어가.
```

## 3. 프로젝트 파일도 함께 첨부하세요.

가능하면 아래 파일도 같이 올리세요.

- 프로젝트 폴더
