# Learning Journey

> 이 문서는 v1 완성 후 작성한 회고입니다. 정확한 과거 순서를 꾸며내기보다, 프로젝트를 진행하며 실제로 발견한 문제와 변경한 판단을 기록합니다.

Zero to Vibe Coding은 완성된 개발 지식을 전달하기 위해 시작한 프로젝트가 아니라, AI와 함께 코딩하며 제가 직접 겪은 이해의 공백을 초보자의 학습 흐름으로 다시 설계한 프로젝트입니다.

## 1. 처음의 접근

- AI에게 만들고 싶은 기능을 설명하고 생성된 명령과 코드를 따라 실행했다.
- 기능은 구현할 수 있었지만 파일이 어디에 생성되는지, 현재 터미널 위치가 왜 중요한지 설명하기 어려웠다.

## 2. 발견한 문제

- 프로젝트 폴더와 일반 폴더의 차이를 이해하지 못했다.
- `cd`를 언제 사용해야 하는지 몰라 다른 위치에서 파일을 실행하는 일이 있었다.
- 터미널, Python, VS Code, Codex의 역할을 하나의 흐름으로 연결하지 못했다.
- 오류가 발생하면 원인을 이해하기보다 AI에게 다음 명령을 다시 요청했다.

## 3. 바꾼 접근

- 기능 개발을 잠시 멈추고 개발환경의 각 요소가 어떤 역할을 하는지 다시 학습했다.
- 한 번에 여러 개념을 배우는 대신 하나의 명령을 실행하고 결과를 확인한 뒤 다음 단계로 넘어갔다.
- 코드보다 먼저 ‘왜 이 작업을 하는지’를 설명하도록 학습 순서를 재설계했다.

## 4. 제품으로 전환한 방법

- 제가 실제로 했던 질문을 초보자의 사용자 요구사항으로 정리했다.
- 각 단계를 ‘왜 하는지-직접 해보기-예상 결과-자주 하는 실수’로 통일했다.
- Python 설치부터 첫 실행, Codex를 통한 수정까지 하나의 학습 흐름으로 연결했다.

## 5. 현재의 한계

- 현재 실습은 macOS 중심이다.
- Windows 사용자 흐름은 충분히 검증하지 못했다.
- 실제 초보자 사용성 테스트가 더 필요하다.

## 6. 다음 검증 계획

- 코딩 경험이 없는 사용자가 별도의 추가 설명 없이 튜토리얼을 수행하도록 한다.
- 중단된 단계, 질문이 발생한 용어, 예상 결과와 달랐던 화면을 기록한다.
- 반복적으로 막힌 단계는 설명과 예시를 수정한다.
